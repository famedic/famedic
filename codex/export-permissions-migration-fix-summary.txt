# Causa exacta del fallo

La migración `database/migrations/2025_07_10_add_export_permissions.php` usaba
`Permission::create()` sin comprobar si el permiso ya existía.

En `migrate:fresh`, los mismos permisos **ya se crean antes** en:

    database/migrations/2024_11_15_024936_create_permission_tables.php

Esa migración lee `config/famedic.php` → clave `permissions` y crea todos los
permisos del árbol, incluyendo los hijos `manage.export` bajo cada categoría.

Error al re-ejecutar `Permission::create()`:

    Spatie\Permission\Exceptions\PermissionAlreadyExists:
    A `laboratory-purchases.manage.export` permission already exists for guard `web`.


# Archivo corregido

| Archivo | Cambio |
|---------|--------|
| `database/migrations/2025_07_10_add_export_permissions.php` | `Permission::firstOrCreate` vía helper `ensureExportPermission()` |

# Dónde se duplicaban

| Origen | Cuándo corre | Qué crea |
|--------|--------------|----------|
| `config/famedic.php` + `2024_11_15_024936_create_permission_tables.php` | Primero (nov 2024) | Los 4 permisos `manage.export` con descripciones del config |
| `2025_07_10_add_export_permissions.php` (antes del fix) | Después (jul 2025) | Los mismos 4 permisos con `Permission::create()` → **colisión** |

Definiciones en `config/famedic.php`:

- `customers` → `['manage.export' => 'Descargar clientes']`
- `laboratory-purchases` → `['manage.export' => 'Descargar ordenes']`
- `online-pharmacy-purchases` → `['manage.export' => 'Descargar ordenes']`
- `medical-attention-subscriptions` → `['manage.export' => 'Descargar membresías médicas']`

No hay seeders que creen estos permisos. `DatabaseSeeder` no toca permisos.


# Cómo se hizo idempotente

Método privado `ensureExportPermission($name, $description, $parentName)`:

```php
Permission::firstOrCreate(
    ['name' => $name, 'guard_name' => 'web'],
    [
        'description' => $description,
        'permission_id' => $parentPermission?->id,
    ]
);
```

- Si el permiso ya existe (p. ej. creado por `create_permission_tables`): **no hace nada**.
- Si no existe (BD antigua migrada antes de añadir `manage.export` al config): **lo crea**
  con `permission_id` del padre.
- No asigna permisos a roles (la migración original tampoco lo hacía).
- Al final se limpia el cache de permisos con el mismo patrón del proyecto:

```php
app('cache')
    ->store(config('permission.cache.store') != 'default' ? config('permission.cache.store') : null)
    ->forget(config('permission.cache.key'));
```
