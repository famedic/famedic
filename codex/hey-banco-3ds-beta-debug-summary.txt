Hey Banco 3DS — Diagnóstico beta + desactivación EfevooPay visible
===================================================================
Fecha: 2026-06-09

RESUMEN EJECUTIVO
-----------------
El checkout NO entraba a 3DS por dos causas combinadas:

1. **payment_method llegaba como ID numérico legacy** (ej. `428`) en lugar de `hey_banco:{id}`.
   - `PaymentMethodIdentifier::isHeyBanco()` exige prefijo `hey_banco:`.
   - IDs numéricos se enrutan a `EfevooPaymentGateway` → `ChargeEfevooPaymentMethodAction`.
   - Si el ID numérico coincidía con `payment_methods.id` de Hey Banco, podía caer en cobro ADQ directo (`token_charge`) sin 3DS.

2. **EfevooPay seguía mezclado en listados** aunque beta apuntara a Banregio.
   - `Customer::paymentMethods()` devolvía tokens Efevoo + Hey Banco.
   - El usuario podía seleccionar tarjeta Efevoo (ID `428`) sin darse cuenta.

Config key correcta para 3DS:
- `config('heybanco.3ds_enabled')` ← mapea `HEYBANCO_3DS_ENABLED` en `config/heybanco.php`
- NO existe `config('heybanco.3ds.enabled')`

Si `.env` cambió pero beta usa `php artisan config:cache`, el valor cacheado puede quedar en `false`.
**Siempre ejecutar tras cambiar .env en beta:**

```bash
php artisan config:clear
php artisan cache:clear
php artisan route:clear
php artisan config:cache   # solo si beta usa config cache en deploy
```

Validar en tinker:

```bash
php artisan tinker --execute="dump(config('heybanco.3ds_enabled'), config('heybanco.enabled'), config('payments.default_provider'), config('payments.efevoopay_enabled'));"
```

DIAGNÓSTICO shouldUseHeyBanco3ds()
----------------------------------
Regresa `true` solo si TODAS se cumplen:
- `config('heybanco.3ds_enabled') === true`
- `PaymentMethodIdentifier::isHeyBanco($paymentMethod)` → prefijo `hey_banco:`
- `amount_cents > 0`

Si llegaba `428` → `is_hey_banco_method=false` → reason=`payment_method_not_hey_banco_prefix`
→ flujo caía en `ChargePaymentMethodAction` → Efevoo o Hey Banco ADQ directo.

PANTALLA ESPERADA SI 3DS SÍ SE ACTIVA
-------------------------------------
1. POST checkout → `HeyBanco3dsRedirectRequiredException`
2. `LaboratoryPurchaseController` → `redirect()->route('payments.hey-banco.3ds.redirect', $session)`
3. Inertia renderiza `HeyBanco3dsRedirect.jsx`
4. `window.location.href` → URL Banregio (`testcolecto`)
5. Callback POST → `payments.hey-banco.3ds.callback`
6. Resultado → `HeyBanco3dsResult.jsx`

BD esperada tras iniciar 3DS:

```sql
select id, folio, status, amount, redirect_url, created_at
from payment_3ds_sessions order by id desc limit 10;

-- status = redirect_required
-- payment_transactions.flow = token_3ds_charge
-- payment_attempts.status = pending_3ds
```

### Tarea B — Desactivar EfevooPay visible
- `config/payments.php` (NUEVO)

  - `PAYMENT_PROVIDER` → `payments.default_provider`
- `app/Models/Customer.php`
  - `paymentMethods()` omite Efevoo si `efevoopay_enabled=false`
- `app/Services/Payments/PaymentGatewayManager.php`
  - Rechaza IDs numéricos si Efevoo deshabilitado
  - `defaultProvider()` prioriza Hey Banco si Efevoo off
- `app/Services/Payments/EfevooPaymentGateway.php`
  - `isEnabled()` lee `payments.efevoopay_enabled`
- `app/Http/Controllers/PaymentMethodController.php`
  - Index/create sin tokens Efevoo si deshabilitado
  - Store rechaza tokenización Efevoo si deshabilitado
- `app/Http/Controllers/LaboratoryCheckoutController.php`
  - Props: `heyBancoEnabled`, `efevoopayEnabled`, `defaultPaymentProvider`
- `resources/js/Pages/PaymentMethods/Create.jsx`
  - Sin radio Efevoo si `efevoopayEnabled=false`

- `resources/js/Components/Checkout/PaymentMethodStep.jsx`
